# Node.js Streams Learning Notes

Welcome to the Stream Alchemy project. This repository contains step-by-step practices designed to master Node.js Streams, buffer management, and backpressure control.

---

## 1. What are Node.js Streams?
Streams are a core Node.js concept used to handle reading/writing files, network communications, or any kind of end-to-end information exchange in an efficient chunk-by-chunk manner.
- Memory Efficiency: Instead of loading an entire large file (e.g., a 1GB log) into memory, data is processed in small pieces (chunks).
- Time Efficiency: Processing can begin as soon as the first chunk is received, rather than waiting for the entire payload to load.

## 2. Progress & Concepts Learned

### Phase 1: The Source (Readable Streams)
* Goal: Create a data source that produces information chunk-by-chunk.
* Key File: [01-the-source/custom-readable.js](file:///Users/rikutomikado/dev/git/stream-alchemy/01-the-source/custom-readable.js)
* Concepts Learned:
  - Subclassing the built-in Readable class.
  - Overriding the internal _read(size) method, which Node.js automatically calls when it needs more data.
  - Using this.push(chunk) to add data into the read queue, and this.push(null) to signal the end of the stream (EOF).
  - Stream consumption via event listeners: data (emitted on every chunk) and end (emitted when no more data is left).

### Phase 2: The Sink (Writable Streams & Backpressure)
* Goal: Create a data destination (sink) and handle flow control (backpressure).
* Key File: [02-the-sink/blackhole.js](file:///Users/rikutomikado/dev/git/stream-alchemy/02-the-sink/blackhole.js)
* Concepts Learned:
  - Subclassing the Writable class and overriding _write(chunk, encoding, callback).
  - Backpressure Management: The writer checks the boolean return value of writable.write(chunk).
    - If write() returns false, it means the internal queue is full (highWaterMark limit reached) and the producer must pause writing to prevent memory bloat.
    - Listening for the 'drain' event to know when the buffer has been cleared and it is safe to resume sending data.

### Phase 3: The Mutator (Transform Streams)
* Goal: Modify or transform data as it passes through the stream.
* Key Files:
  - [03-the-mutator/censor-stream.js](file:///Users/rikutomikado/dev/git/stream-alchemy/03-the-mutator/censor-stream.js)
  - [03-the-mutator/csv-to-json.js](file:///Users/rikutomikado/dev/git/stream-alchemy/03-the-mutator/csv-to-json.js) (Pending)
* Concepts Learned:
  - Subclassing the Transform class (which is both Readable and Writable).
  - Overriding _transform(chunk, encoding, callback) to process the data chunk-by-chunk.
  - Implementing an anonymizer/censor logic that filters out sensitive words (e.g., replacing 'ERROR' with '[REDACTED]') before pushing data downstream.
  - Overriding _flush(callback) to append final operations right before the stream closes.
